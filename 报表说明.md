# 报表说明 — 指标定义与口径

> 直播间销售数据分析仪表盘（11 个 Tab）的指标定义、计算口径与注意事项。
> 看数前先读「全局概念」，再看对应 Tab。

---

## 一、全局概念（所有 Tab 通用）

### 数据来源
- **正装订单**：`报表订单.xlsx`（含 23-24 / 25 / 26 三个 sheet），是绝大多数 Tab 的数据底座。
- **派样 / 会员 / 人群**：分别来自 `派样.xlsx` / `会员.xlsx` / `人群.xlsx`，对应各自的 Tab。
- 所有金额口径 **GMV = 订单应付金额（实付）**，不是商品原价。

### 左侧「全局筛选」（影响除特别说明外的所有 Tab）
| 筛选 | 含义 |
|---|---|
| **时间范围** | 统计的订单支付时间区间。 |
| **开启同比(YOY)** | 勾选后用「同比时间段」（默认主区间往前推 1 年）做对比，KPI 显示差值。 |
| **渠道类型 / 达人 / 品类 / 货号 / 订单状态** | 按对应字段过滤订单；不选 = 全选。 |

### 新客 / 老客判定（可在左侧「⚙️ 老客判定规则」调整）
- **老客**：该用户在当前订单之前（至少 **1 天**）有过 **≥ 550 元**、状态属于 **成交（已完成 / 已发货 / 待发货，排除已关闭）** 的记录。
- **新客**：不满足上述条件的订单。
- 默认用 **R12 滚动口径**（窗口默认 9999 天 ≈ 全时段），可在侧边栏改 有效成交渠道 / 金额 / 天数 / 窗口。
- **人数按「互斥」口径（新客优先）**：一个用户只要有任意一笔订单被判为新客，就整体算**新客**；否则算老客。所以 **新客人数 + 老客人数 = 去重买家数**（顶部 KPI、渠道汇总、新老客趋势、货品分析 全部一致）。订单数 / GMV 是订单级，不受此影响。

#### 改「老客判定规则」会联动哪些 Tab？
改 4 个参数（有效成交渠道 / 最低金额 / 早于N天 / R12回溯窗口）任一 ≠ 默认值，会**重算新老客标签**并同步影响：

| 跟随情况 | 模块 |
|---|---|
| ✅ **完全跟随** | 表头 KPI、渠道汇总、新老客趋势、货品分析、复购率、复购周期、派样（新客人数/拉新率）|
| ⚠️ **部分跟随** | RFM —— 只跟「最低金额」（老客基数门槛用它）；**不跟** 有效成交渠道 / 早于N天 / R12窗口（RFM 基数=截止日前买过≥550成交，渠道由 RFM 自己的「老客基数渠道」控制）|
| ❌ **不跟随** | 渠道流转（纯渠道流转不分新老客）、会员（按「入会时间」分新老会员，另一套定义）、人群（无新老客拆分）、平台优惠 |

### 顶部 KPI 卡片
| 指标 | 定义 |
|---|---|
| 订单数 | 筛选范围内订单行数。 |
| GMV | 订单应付金额合计。 |
| 去重买家数 | 不重复的淘宝 ID 数。 |
| 购买频次 | 订单数 ÷ 去重买家数。 |
| 客单价 | GMV ÷ 去重买家数。 |
| 新/老客占比 | 按上面的「按人」口径拆分。 |

## 二、各 Tab 指标定义

### 🏷 渠道汇总
按**渠道类型 / 达人**等维度汇总：订单数、GMV、人数，并拆新老客（新客订单/GMV/人数、占比、客单）。开 YOY 时附同比列。

### 📈 新老客趋势
按天/按月看**新客 vs 老客**的订单数 / GMV / 人数趋势；下方汇总表给区间合计；开 YOY 叠加去年同期曲线。

### 📦 货品分析
按**货号**排名：订单数、购买人数、GMV、客单。用于看哪些品卖得好、谁在买。

### 📐 RFM
- **回购窗口 = 左侧时间范围**；**分析截止日 = 时间范围开始日的前一天**，R 从此日往前推。
- **老客基数 = 截止日前「购买过」（≥550 且成交：已完成/已发货/待发货）的去重用户**，每人按最近一次购买落入一个 R 桶、只算一次，各桶相加 = 基数总数。R=最近购买距截止日天数；F=截止日前累计订单数；M=截止日前累计 GMV。
- **回购率 = 该桶里在回购窗口内、且符合左侧全局筛选又购买的比例**。
- **两个渠道口径分工**：
  - **「老客基数渠道」**（RFM 内）只决定**基数池**是哪些渠道的客户（不选=全部，独立于左侧全局渠道）。
  - **回购**走**左侧全局筛选**（如自营）。
  - 👉 想让**回购人数 ≈ 表头老客**：左侧全局选某渠道（如自营）+「老客基数渠道」选**全部**（因为表头按全渠道历史判定老客）。两者会几乎逐人对上（差几个是「本期内才第一次复购」的窗口边角）。
- 提供 R / F / M 各维度分布，以及 R×F 交叉透视。

### 🔄 渠道流转（Sankey）
基于「相邻两次购买」的配对，看用户从**上一次渠道/达人**流向**下一次渠道/达人**的路径。低于 `SANKEY_MIN_COUNT`(=3) 次的路径不显示。

### 🔁 复购率
- **回购窗口 = 左侧时间范围**；**分析截止日 = 时间范围开始日的前一天**。
- **老客基数**：截止日之前有过购买的用户，按 R/F/M 分桶。
- **回购**：该用户在「截止日 ~ 回购窗口结束」之间又有购买。
- **回购率 = 回购人数 ÷ 老客基数**，并给 R×F 交叉透视。

### ⏱ 复购周期
基于相邻两次购买的时间间隔，看**两次购买之间隔多少天**的分布（中位数/均值/分布直方图）。

### 🧪 派样（试用装转正装）
> 见下面「三、派样模块详解」。

### 🪪 会员
- 会员表仅含「淘宝 ID + 入会时间」。
- 按**入会时间**区分新/老会员，与订单匹配看其购买表现；并做**会员 vs 非会员**对比。
- 购买口径 = 左侧全局筛选。

### 👥 人群（上传人群匹配）
- 上传人群（人群名称 + 淘宝 ID）与订单匹配，看各人群后续购买。
- **购买转化率 = 购买人数 ÷ 人群去重人数**（购买在全局筛选范围内）。
- 含**各人群对比表**、**人群购买趋势图**（可切按天/按月，看 GMV/订单/人数变化）、**人群购买货品**排名。

### 💰 平台优惠
解析订单里的平台补贴券「券名-金额」，统计各券的使用次数与补贴金额。

## 三、派样模块详解（🧪 派样）

**目的**：看买过低价试用装（派样）的顾客，后续有没有转化成正装购买。

**回购定义**：在所选派样窗口内买过试用装的顾客，只要在**左侧全局筛选范围内有过正装购买**即算回购（不要求正装购买时间晚于买样时间）。

**回购观察口径 = 左侧全局筛选**；模块内的四个筛选只作用于派样（活动期间）：
| 筛选 | 作用 |
|---|---|
| ① 派样购买时间 | 圈定「活动期间」，定义哪些买样人进入分析。 |
| ② 派样订单状态 | 按派样订单状态过滤。 |
| ③ 派样类型 | 付邮试用 / 尝鲜礼盒 等（源表「派样类型」列）。 |
| ④ 派样货号 | 只列出**所选时间窗内出现过**的货号（随派样类型联动）。 |

### 顶部 KPI
| 指标 | 定义 |
|---|---|
| 派样订单数 | 所选派样窗口内的派样订单总数（含同人多单，不去重）。 |
| 买样人数 | 去重的买样淘宝 ID 数。 |
| 新客人数 | 买样用户中、在全局筛选内有正装订单且被判为**新客**的人数（口径同全店 KPI，受左侧老客判定规则影响）。 |
| 拉新率 | 新客人数 ÷ 买样人数。 |
| 回购人数 | 买样用户中有正装购买的去重人数。 |
| 回购订单数 | 这些人的正装订单数（不去重）。 |
| 回购率 | 回购人数 ÷ 买样人数。 |
| 回购GMV | 这些正装订单的 GMV 合计。 |
| （开 YOY）差值 | 同一批买样用户在「同比时间段」内的正装回购表现差值。 |

### 派样货品明细（按 派样类型 × 货号）
| 列 | 定义 |
|---|---|
| 派样订单数 | 该货品的派样订单数（不去重）。 |
| 派样人数 | 该货品的去重买样人数。 |
| 回购人数 | 这些人中有正装购买的去重人数。 |
| 回购订单数 | 这些人的正装订单数（不去重）。 |
| 回购金额 | 这些正装订单的 GMV 合计。 |
| 回购AUS | 回购金额 ÷ 回购人数。 |
| 回购率 | 回购人数 ÷ 派样人数。 |

> 同一人若试用多款，会分别计入各款；每个派样类型的「小计」行按人去重。

---

_口径若与公司系统有出入，以本说明 + `config.py` 中的常量为准（如 老客金额 `OLD_CUSTOMER_MIN_AMOUNT=550` / 有效成交状态 `TRANSACTION_SUCCESS_STATUSES=已完成,已发货,待发货`）。_
