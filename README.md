# 直播间销售分析仪表盘 — Windows 部署指南

> 适用于公司 Windows 电脑首次部署。Mac 版请看 `PROJECT_STATUS.md`。

---

## 一、首次安装（只做一次，约 5-10 分钟）

### 第 1 步：安装 Python

1. 打开 https://www.python.org/downloads/windows/
2. 推荐下载 **Python 3.12.x → Windows installer (64-bit)**（最稳，所有依赖都有现成 wheel）
   - 3.13.x 也能用，但部分包装得慢或要现编译
   - 不要用 3.10 以下，新版 streamlit 不支持
3. 安装时 ✅ **务必勾选 `Add python.exe to PATH`**（在安装窗口最底部）
4. 安装完成后，按 `Win+R` → 输入 `cmd` → 回车，运行：
   ```
   python --version
   ```
   能看到 `Python 3.10.x` 之类输出即成功。

### 第 2 步：解压本文件包

把 `xinlaoke-dashboard.zip` 解压到任意位置，例如：
```
D:\xinlaoke\
```
**不要解压到 C:\Program Files\** 等需要管理员权限的目录。

### 第 3 步：放入数据源表

把 **`报表订单.xlsx`** 放进 `data\raw\` 目录：
```
D:\xinlaoke\data\raw\报表订单.xlsx
```
> Excel 必须包含 3 个 sheet：`23-24`、`25`、`26`，列名固定（淘宝ID / 货号 / 订单状态 / 平台优惠 等）。

**可选源表**（按文件名关键词识别，放进同一个 `data\raw\` 即可，不放则对应 Tab 显示"尚未生成数据"）：

| 文件名（含关键词） | 列 | 对应 Tab |
|---|---|---|
| `派样.xlsx` | 与报表订单同结构 + **派样类型**列(货号前) | 🧪 派样 |
| `会员.xlsx` | 淘宝ID、入会时间 | 🪪 会员 |
| `人群.xlsx` | 人群名称、淘宝ID | 👥 人群 |

> 多张表都放 `data\raw\`，预处理按文件名关键词（报表订单/派样/会员/人群）各认各的，不会混。

### 第 4 步：双击 `启动仪表盘.bat`

首次运行会自动：
1. 安装依赖（streamlit / pandas / plotly 等，约 1-3 分钟）
2. 预处理 Excel → parquet（约 1-3 分钟，看数据量）
3. 启动 Streamlit 服务

启动成功后，会自动弹出浏览器并显示：
- 本机链接：`http://localhost:8501`
- 内网链接：`http://<你的内网IP>:8501`

## 二、日常使用

直接双击 `启动仪表盘.bat` 即可。已经预处理过的话，几秒内就启动。启动后 cmd 窗口会列出三种访问方式 + 网页密码。

### 🔑 访问密码
所有链接打开后都要先输**网页密码**，默认 `xinlaoke2026`。
- 改密码：用记事本打开 `启动仪表盘.bat`，改最上面的 `set "DASH_PASS=xinlaoke2026"` 这一行（留空 = 不要密码）。

### 让同事访问（两种方式）
| 方式 | 适用 | 怎么发 |
|---|---|---|
| **内网链接** | 同公司网络的同事 | 把窗口里的「内网链接」（如 `http://172.16.13.113:8501`）发给对方 |
| **🌍 公网链接** | 任意外网的人（在家、外地、手机） | 把窗口里的「公网链接」（`https://xxxx.trycloudflare.com`）发给对方 |

- 公网链接由 **cloudflared** 自动开通，**对方什么都不用装**，浏览器打开输密码即可。
- ⚠️ **公网链接每次启动会变**，重启后请把窗口里的最新网址重新发给同事。
- ⚠️ 你的电脑 + 这个 cmd 窗口要保持开着，关掉服务就停（对方就打不开了）。

> 📖 各报表的指标定义、计算口径见 **`报表说明.md`**。

## 三、更新数据（换新的 Excel）

> ⚠️ **关键：仪表盘只读 `data/processed/` 里的 parquet，不直接读 Excel。**
> 换了新 Excel 后，**必须重新跑一次预处理**，把 xlsx 转成 parquet，页面才会更新。
> 只把新 Excel 放进 `data/raw/` 但不跑预处理 → 页面数据不变（甚至报「数据文件不存在」）。

1. 把新的 `报表订单.xlsx` **覆盖** `data\raw\` 下的旧文件（先确认 Excel 已关闭，否则会有 `.~xxx.xlsx` 临时锁文件，读取可能报错）
2. 重新跑预处理（二选一）：
   - 双击 `启动仪表盘.bat`，会自动重新预处理；**或**
   - 命令行手动：
     ```
     cd D:\xinlaoke
     python preprocess.py
     ```
3. 刷新浏览器页面（⌘R / Ctrl+R）即可看到新数据

> 🛑 **不要删 `data/raw/` 或 `data/processed/` 文件夹。** 这两个目录被 `.gitignore` 排除、不在 git 里，删了无法用 `git` 恢复，只能重新放 Excel + 重跑预处理。做「备份清理」时务必跳过 `data/` 目录。

## 四、内网访问常见问题

### ❓ 同事访问不了

依次检查：

1. **本机能访问吗？**
   浏览器打开 `http://localhost:8501`，能看见仪表盘 → 服务正常。
   看不到 → 启动脚本有报错，看 cmd 窗口输出。

2. **同事跟你在同一个内网吗？**
   你的电脑显示的内网 IP（如 `172.16.13.113`）必须和同事在同一个网段（前三段相同）。
   不同楼层、不同子网、走 VPN 都可能不通。

3. **Windows 防火墙拦了吗？**
   首次启动 Python/streamlit 时，Windows 防火墙会弹窗 → **勾选「专用网络」+「公用网络」→ 允许访问**。
   如果错过了弹窗：
   - `控制面板` → `Windows Defender 防火墙` → `允许应用通过防火墙`
   - 找到 `Python` → 勾选「专用」和「公用」
   - 或者直接关闭防火墙测试（仅排查用）

4. **端口 8501 被占用？**
   命令行运行：
   ```
   netstat -ano | findstr 8501
   ```
   如果有其他进程占用，要么杀掉它，要么改启动脚本里的端口号。

5. **电脑睡眠了？**
   电脑睡眠或关机后服务停止。建议：
   - 控制面板 → 电源选项 → 选择「高性能」
   - 「关闭显示器」可以，但「使计算机进入睡眠」选「从不」

### ❓ IP 经常变？

公司路由器一般是 DHCP 动态分配 IP，今天 `.113` 明天可能变 `.115`。两种解决：

1. **找 IT 做 DHCP 绑定**（推荐）：告诉 IT 你电脑的 MAC 地址（命令行 `ipconfig /all` 查），要求每次都分配 `172.16.13.113`。
2. **手动设静态 IP**：网络设置 → 以太网/WiFi 属性 → IPv4 → 改成手动指定，IP 填 `172.16.13.113`，注意别和别人冲突。

## 五、文件结构

```
xinlaoke/
├── app.py                  # Streamlit 主应用（11个Tab）
├── config.py               # 字段名 & 基础参数（含多表关键词）
├── preprocess.py           # Excel → Parquet 预处理
├── requirements.txt        # Python 依赖列表
├── 启动仪表盘.bat          # Windows 双击启动（含 cloudflared 公网隧道）
├── 启动仪表盘.command      # macOS 双击启动（公司用Win可忽略）
├── cloudflared.exe         # 公网隧道程序（Windows，随包附带，勿删）
├── README.md               # 本文件（部署 + 使用）
├── 报表说明.md             # 各 Tab 指标定义与口径
├── PROJECT_STATUS.md       # 项目完整文档（含数据结构、Tab 说明）
├── data/
│   ├── raw/                # 放源表：报表订单.xlsx（必需）；派样.xlsx / 会员.xlsx / 人群.xlsx（可选）
│   └── processed/          # 预处理生成（首次启动自动创建）
└── .streamlit/
    └── config.toml         # 主题配置
```

## 七、新老客判定规则（侧边栏可改）

| 参数 | 默认 | 说明 |
|------|------|------|
| 有效成交渠道 | 全渠道 | 只有"自营"才算有效成交（可选） |
| 最低金额 | ¥550 | 低于此金额不算有效成交 |
| 最少间隔天数 | 1 天 | 历史订单需早于当前订单至少 N 天 |
| R12 回溯窗口 | 9999 天 | ≈ 全时段 |

**老客定义**：当前订单时间 ≥ 该用户首次「已完成 且 金额≥550」订单时间 + 1 天，否则为新客。
